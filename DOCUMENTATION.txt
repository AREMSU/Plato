  - babel.config.js
- Android native project:

- App source code:
  - src/components/*
  - src/screens/*
  - src/navigation/*

  - src/data/*
  - src/utils/*
- Existing assets folder:

Branding state before updates:

## 3) Notes

- Android manifest already points launcher icons to mipmap resources, so icon replacement is active for Android builds.
- Existing .gitignore at root and android/.gitignore remain suitable for these assets (images are not ignored).
- A validation warning exists in app.json about newArchEnabled in the current schema tooling; this key existed in the project and was not introduced by this branding change.

## 4) How To Update Branding Later

1. Replace files in assets/branding with new versions using same names.
2. Regenerate Android mipmap launcher files if shape/style changes.
3. Keep splash logo at android/app/src/main/res/drawable-nodpi/splash_logo.png or replace with same filename.
4. Rebuild Android app to verify launcher + splash updates.
